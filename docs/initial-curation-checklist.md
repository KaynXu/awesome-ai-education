# Maintainer Curation Checklist

Use this checklist for every suggestion, pull request, promotion, or review.

## Scope Screen

- [ ] The resource supports AI for educational practice, not education about AI.
- [ ] The target user is explicit.
- [ ] The education workflow is concrete.
- [ ] The role of AI is explained without marketing language.
- [ ] The primary public URL is stable and inspectable.

## Evidence Review

- [ ] The strongest public source is linked directly.
- [ ] The evidence owner is identified as independent, institution-published, vendor-published, open source, or product documentation.
- [ ] Vendor-reported claims are labeled.
- [ ] The editorial note states what the source does not establish.
- [ ] The submitter affiliation and self-promotion disclosure are recorded.
- [ ] The evidence record includes a `Last reviewed` date.

## Editorial Decision

- [ ] The selected layer is Main List, Open Source, Field Evidence, or Watchlist.
- [ ] No duplicate project entry exists in another workflow section.
- [ ] External links were opened during review.
- [ ] The wording names workflow, target user, and evidence type.
- [ ] A human maintainer approved the decision.

## Bilingual Merge

- [ ] The English entry is in the intended editorial order.
- [ ] The Chinese README has the same project name, URL, evidence link, layer, and relative order.
- [ ] The English and Chinese H2 structures remain aligned.
