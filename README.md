## Teaching and Lesson Design

- [MagicSchool](https://www.magicschool.ai/) - Helps teachers and school teams create lessons, assessments, accommodations, communications, and student activities; [independent evaluation](docs/evidence.md#magicschool).
- [Eduaide](https://www.eduaide.ai/) - Helps teachers design lessons, assessments, differentiated materials, and standards-aligned resources; [vendor case study](docs/evidence.md#eduaide).
- [Curipod](https://curipod.com/) - Helps teachers run interactive lessons and writing practice with immediate student feedback; [vendor case study](docs/evidence.md#curipod).
- [Teachmate](https://www.teachmate.com/) - Helps teachers plan, adapt resources, write reports, and reduce communication workload; [vendor case study](docs/evidence.md#teachmate).

## Assessment and Feedback

- [Brisk Teaching](https://www.briskteaching.com/) - Helps teachers create feedback, assessments, differentiated materials, translations, and student activities inside existing browser workflows; [vendor case study](docs/evidence.md#brisk-teaching).
- [Gradescope](https://www.gradescope.com/) - Helps instructors grade paper, online, and programming assignments with answer grouping, rubrics, and reusable feedback; [public deployment](docs/evidence.md#gradescope).
- [Writable](https://www.writable.com/) - Helps teachers manage writing assignments, rubrics, feedback, and revision cycles; [product documentation](docs/evidence.md#writable).

## Tutoring and Learner Support

- [CENTURY Tech](https://www.century.tech/) - Gives students adaptive practice paths and helps teachers identify curriculum gaps; [independent evaluation](docs/evidence.md#century-tech).
- [Flint](https://flintk12.com/) - Lets K-12 teachers assign AI-supported activities and review student conversations and feedback in a managed environment; [product documentation and external review](docs/evidence.md#flint).

## Content and Curriculum

- [Merlyn](https://www.merlyn.org/) - Helps teachers control classroom technology, present content, search, and use a school-managed voice assistant; [independent evaluation with vendor-hosted summary](docs/evidence.md#merlyn).

## Teacher Development and Coaching

- [Edthena AI Coach](https://www.edthena.com/ai-coach-for-teachers/) - Guides teachers through structured reflection and instructional coaching cycles using classroom evidence; [research record](docs/evidence.md#edthena-ai-coach).

## Open Source

- [Shiori](https://github.com/kaorii-ako/Shiori-v1) - Gives learners Google Classroom sync, study plans, quizzes, and spaced-repetition flashcards in an MIT-licensed, self-hostable study companion; [open source review](docs/evidence.md#shiori).

Open source is a distinct editorial layer, not a requirement for the whole guide. Code availability makes a project inspectable, but it does not prove educational impact.

## Field Evidence

- [Ottawa Catholic School Board + Brisk](https://www.briskteaching.com/success-stories/ottawa-catholic-school-board) - Describes a district-scale rollout inside Google Workspace for feedback, personalized support, and multilingual access. Vendor-reported case study.
- [Barbers Hill ISD + Brisk](https://www.briskteaching.com/success-stories/barbers-hill-independent-school-district) - Describes district adoption for targeted feedback, student activities, translation, text leveling, and differentiated support. Vendor-reported case study.
- [Ysleta Middle School + Curipod](https://curipod.com/case-studies/curipod-case-study-ysleta-bilingual) - Describes writing feedback and classroom participation with emergent bilingual learners. Vendor-reported case study.

These records show implementation settings, not independent endorsements. Vendor ownership is stated so readers can judge each source appropriately.

- [QuizVerse](https://quizverse.world/) - Offers adaptive exam practice, automated grading, and multilingual feedback for learners; [awaiting public deployment evidence](docs/evidence.md#quizverse).
- [Kira Learning](https://www.kiralearning.com/) - Combines computer science curriculum, classroom management, and AI-supported learning; [external review exists, but outcome evidence remains limited](docs/evidence.md#kira-learning).
- [Cognii](https://www.cognii.com/) - Provides conversational tutoring and feedback on open responses; [awaiting a current named deployment and recent evaluation](docs/evidence.md#cognii).

- [Mindsmith](https://www.mindsmith.ai/) - Helps learning designers create lessons, training modules, assessments, and shareable e-learning content; [awaiting non-vendor evidence](docs/evidence.md#mindsmith).

Watchlist is an active review queue, not a lower-quality directory. A resource moves only when public evidence changes.

## How Curation Works

Every Main List decision is made by a human maintainer. AI may help discover candidates, gather sources, or draft copy, but it may not approve an entry.

Each selected resource must show a concrete education workflow, a clear target user, and public evidence. Evidence labels describe source ownership and strength without turning uncertain evidence into a numeric score. Vendor case studies stay visibly vendor-reported, and resources move between Main List and Watchlist as evidence changes.

The public [evidence ledger](docs/evidence.md) records sources, limitations, and review dates. The full [curation criteria](docs/criteria.md) define admission, movement, and removal. General AI tools, AI-literacy courses, and paper-only collections stay out unless they connect to a practical education implementation.
